# ush Start

`ush` has two sides:

- `ush_shell` for interactive shell usage
- `ush_compiler` for compiling `.ush` into portable POSIX `sh`

The core idea is simple: even `.ush` scripts still end up running through `/bin/sh`.
`ush` tries to stay close to Unix and POSIX instead of building a large custom runtime.

> `ush` is still an early WIP, and this guide focuses only on the parts that already work today.
## 1. Start the shell

During development:

```bash
cargo run -p ush
```

To try a one-liner:

```bash
cargo run -p ush -- -c 'printf "a\nb\n" | len'
```

If `ush` is already on your `PATH`:

```bash
ush
ush -c 'printf "ok\n"'
```
## 2. Use it like a normal shell

Simple POSIX-style commands work as expected:

```bash
ush -c 'pwd'
ush -c 'printf "hello\n" | grep h'
ush -c 'if true; then printf "ok\n"; fi'
```

More complex POSIX syntax may fall back to `/bin/sh`.
## 3. Use structured helpers

One of the most interesting parts of `ush` is that you can add structured helpers to normal text pipes.

```bash
printf "a\nb\nc\n" | len
printf "hello\nworld\n" | map(\it -> upper(it))
printf "hello\nworld\n" | fmap(\it -> upper(it))
printf "hello\n" | map(\line -> { upper(line) })
printf "x\ny\n" | map(\-> { "ok" })
printf "a\nb\nc\n" | car
printf "a\nb\nc\n" | cdr
printf "a\nb\nc\n" | take(2)
printf "a\nb\nc\n" | drop(1)
printf "a\nb\nc\n" | nth(1)
printf "a\nb\nc\n" | enumerate(1)
printf "b\na\nb\n" | fsort | funiq | fjoin(",")
printf "a\nb\nc\n" | flat(\head, rest -> [head, "tail", rest])
printf "a\nb\n" | fzip(["1", "2"])
printf "a\nb\n" | fzip(["1", "2"]) | swap
printf "a\nb\n" | fzip(["1", "2"]) | fst
```

Common helpers:

- `len`
- `lines`
- `json`
- `xml`
- `html`
- `car`
- `cdr`
- `head`
- `tail`
- `take(...)`
- `drop(...)`
- `nth(...)`
- `enumerate(...)`
- `swap`
- `fst`
- `snd`
- `frev`
- `fsort`
- `funiq`
- `fjoin(...)`
- `map(...)`
- `fmap(...)`
- `flat(...)`
- `ffmap(...)`
- `fzip(...)`
- `each(...)`
- `filter(...)`
- `ffilter(...)`
- `any(...)`
- `fany(...)`
- `some(...)`
- `fsome(...)`

`length` still works as a compatibility alias, but `len` is the preferred name now.

### Helper lambda syntax

Helper lambdas currently support:

```text
\it -> upper(it)
\line -> { upper(line) }
\-> { "ok" }
\head, rest -> [head, rest]
```

At the moment, this lambda syntax is for stream helpers only.
It is not yet a full first-class closure system for the whole `.ush` language.

## 4. `json`, `xml`, and `html`

JSON:

```bash
cat package.json | json | len
```

XML:

```bash
cat feed.xml | xml
```

HTML:

```bash
curl -fsSL https://example.com | html
```

Current behavior:

- `json`: parse as JSON when possible, otherwise fall back to the browser flow
- `xml`: pretty-print valid XML, otherwise fall back to the browser flow
- `html`: write a temporary HTML file and open it in the default browser

## 5. Write a `.ush` script

A tiny script looks like this:

```ush
let greeting = "hello"
print greeting + " world"
shell "printf '%s\n' from-ush"
```

Run it:

```bash
ush examples/hello.ush
```

If you want to inspect the compiled shell output:

```bash
ush compile examples/hello.ush
```

## 6. Typed functions

Functions use Rust-like signatures:

```ush
fn greet(name: String, count: Int) -> String {
  return name + ":" + count
}

print $ greet "ush" 2
print $ greet count: 3 name: "mini"
```

Labeled arguments are supported.
Parameters with `#[default(...)]` can be omitted.

```ush
fn greet(name: String, #[default(2)] count: Int) -> String {
  return name + ":" + count
}
```

## 7. `type`, `enum`, and `match`

Right now, `type` is struct-like sugar.

```ush
type User {
  name: String,
  age: Int,
}

let user = User { name: "ush", age: 7 }
match user {
  User { name, age } => print name + ":" + age
  _ => print "fallback"
}
```

Nested `enum` and `match` are also supported.
The fastest way to see that style is [adt.ush](/Users/nishimura/Code/github.com/ubugeeei/ubshell/examples/adt.ush).

## 8. `async` / `await`

`async` currently exists, but it is subprocess-based rather than green-thread based.

```ush
fn worker(message: String) -> String {
  return message
}

let task = async worker "worker"
let result = task.await
print result
```

You can also spawn a block directly:

```ush
let task = async {
  let prefix = "work"
  prefix + "er"
}
print task.await
```

See [async.ush](/Users/nishimura/Code/github.com/ubugeeei/ubshell/examples/async.ush) and [async_block.ush](/Users/nishimura/Code/github.com/ubugeeei/ubshell/examples/async_block.ush) for working examples.

## 9. Build a CLI with `bin.ush`

In `bin.ush`, the `bin(...)` function becomes the entrypoint.

```ush
fn bin(#[alias("n")] name: String, #[default(2)] count: Int, verbose: Bool) {
  print name + ":" + count
  print verbose
}
```

Run it like this:

```bash
ush examples/bin.ush --name ush --count 4 --verbose
ush examples/bin.ush -n mini
```

That signature is used to generate:

- `--name`
- `--count`
- `--verbose`
- `-n`
- completion candidates for `--complete`

## 10. Documentation comments

`#|` comments are used for generated `--help`, `--man`, and `--complete`.

```ush
#| Demo script.
#| @usage docs.ush --man greet
#| Greet a user.
fn greet(name: String) -> String {
  return "hi " + name
}
```

Try it:

```bash
ush examples/docs.ush --help
ush examples/docs.ush --man greet
ush examples/docs.ush --complete gr
```

## 11. Test scripts

`ush test` treats each `.ush` file as one test process and reports pass/fail from the exit status.

```bash
ush test examples/smoke_test.ush
ush test tests
```

See [smoke_test.ush](/Users/nishimura/Code/github.com/ubugeeei/ubshell/examples/smoke_test.ush) for a tiny passing example.

## 12. Current limitations

- `.ush` is not finished yet
- this is not a fully native POSIX shell implementation; some syntax still falls back to `/bin/sh`
- helper lambda syntax exists, but general first-class anonymous functions are still incomplete
- HM inference, generics, HKT, user-defined modules, `yield`, and green threads are still in progress
- stylish output belongs to the interactive shell side, not to compiled `.ush` stdio behavior
